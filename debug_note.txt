常見原因

- 專案內把連結寫到錯誤的 /docs/ 路徑。

解法
- GitHub Repo → Settings → Pages
  - Source: Deploy from a branch
  - Branch: main
  - Folder: /docs
- 正確站點根網址：
  - https://richardc5168.github.io/ai-math-web/
  - 線性：/linear/
  - 二次：/quadratic/

三、返回鍵 / 根目錄連結造成的導覽問題
症狀
- 使用者點到 repo root（/ai-math-web/）後，可能因為 location.replace / meta refresh 導致返回鍵體驗不佳。

原因
- HTML 內出現指向 repo root 的硬連結，例如：
  - https://richardc5168.github.io/ai-math-web/
  - href="/ai-math-web/"

解法 / 防呆
- 頁面內要回主頁請用相對路徑（例如 `../`）或指向 docs dashboard。
- commit 前 checklist 會掃描 docs 與 dist 的 html，若發現上述字串就 fail。

四、pre-commit 無法安裝 / 沒有生效
症狀
- `pre-commit install` 失敗，或 commit 時沒有跑檢核。

原因
- git 設定了 `core.hooksPath=.githooks`，pre-commit 會拒絕直接安裝到 .git/hooks。

解法
- 本 repo 已經把 hook 直接放在：
  - `.githooks/pre-commit`：跑嚴格檢核
  - `.githooks/prepare-commit-msg`：要求輸入 YES
  - `.githooks/post-commit`：印出 web 連結

五、commit 前建議的嚴格檢核流程（手動跑）
- `C:/Users/Richard/Documents/RAGWEB/.venv/Scripts/python.exe scripts/verify_all.py`
- `C:/Users/Richard/Documents/RAGWEB/.venv/Scripts/python.exe scripts/run_checklist.py --spec tests/specs/precommit_checklist.json`
- `C:/Users/Richard/Documents/RAGWEB/.venv/Scripts/python.exe -m pre_commit run --all-files`

預設流程（已自動化）
- 直接執行 `git commit` 時，會由 `.githooks/pre-commit` 自動跑：
  - pre-commit hooks（whitespace/json/yaml/merge conflict）
  - strict checklist（docs/dist 同步、API smoke、forbid patterns、offline-math 黑屏防呆、debug_note 必備）
- 若任何一項 FAIL，commit 會被阻擋，直到修正後重新 commit。

六、回朔前一版（保留彈性）
推薦：git revert（安全、保留歷史）
- 查 commit：`git log --oneline -n 20`
- 回朔某個 commit（建立反向 commit）：`git revert <commit_sha>`
- 推上去：`git push origin main`

需要直接回到某個版（較危險，會改寫歷史；除非你確定）
- `git reset --hard <commit_sha>`
- `git push -f origin main`

解法

八、offline-math 畫面出不來 / 卡在「載入中…」
症狀
- 打開 https://richardc5168.github.io/ai-math-web/offline-math/?v=2
  - UI 可能出現，但題目一直停在「載入中...」
  - 或直接黑屏 / 顯示載入超時

常見原因（曾經發生過）
- 依賴的 JS 使用了較新的語法（尤其 `?.` optional chaining、或 object spread），在某些較舊/內嵌瀏覽器會直接「語法錯誤」，導致整頁 JS 不執行。
- bank.js 載入方式不穩定（曾用 fetch + Function/eval 類方式），在某些環境會被擋或行為不一致。

解法 / 防呆（本 repo 已採用）
- offline-math 改為用 `<script src="./bank.js?...">` 直接載入題庫（避免 eval）。
- offline-math 使用相容版統計腳本（`aggregate_compat.js`），並避免在關鍵流程使用 `?.`。

commit 前必跑驗證（已納入 strict checklist）
- 確認 offline-math 必備檔存在：
  - `docs/offline-math/bank.js`
  - `docs/offline-math/aggregate_compat.js`
- 確認 offline-math 頁面有引用上述檔案（contains 檢查）。
- 禁止在 offline-math 關鍵腳本出現 `?.`（避免語法錯誤黑屏）。
